# YuMark v0.7

## 更新概述

v0.7 是一次以 AI Agent 能力、AI 编辑安全性和稳定性为核心的版本更新。这个版本让 YuMark 的 AI 助手从简单的对话式响应，升级为可规划、可观察、可阻塞处理的文档任务执行流程，同时补强图片附件、差异预览和多模型工具调用兼容性。

## 核心亮点

### 1. Agent V2 任务运行时

- 新增 Agent 任务、步骤和证据记录模型。
- Agent 会先规划任务，再按步骤执行，而不是只依赖单轮回复。
- 执行过程中可以看到任务目标、当前步骤、步骤状态和阻塞原因。
- 工具执行失败时会进入明确的阻塞状态，避免一直循环或静默失败。
- 用户手动停止 Agent 时，任务进度会正确结束并标记为已阻塞。

### 2. 更安全的 AI 文档编辑

- 创建和编辑文档仍然需要用户确认后才会真正写入。
- 写入类工具会生成待批准的操作提议，不会直接覆盖文档。
- 编辑已有文档时支持差异预览和逐块确认，降低误改风险。
- 创建文档时不再因为缺少保存位置、标题、结构或深度而反复追问，AI 会根据主题自行补全草稿方案。

### 3. AI 图片附件支持

- AI 对话支持添加图片附件。
- 图片会在发送前进行校验、压缩和保存，避免直接把大图塞进上下文。
- 支持多模态模型读取图片内容，用于截图分析、图片说明、视觉资料整理等场景。

### 4. 工具调用和模型兼容性增强

- 改进 OpenAI / Claude / Gemini 等适配器的工具调用消息格式。
- 新增流式输出重试和错误映射，减少网络抖动或模型返回异常造成的中断。
- 增强工具调用参数累积，降低流式 function calling 参数不完整的问题。
- 对工具结果做上下文预算控制，避免长文档或大结果撑爆模型上下文。

### 5. 文件与编辑体验改进

- 增强文档搜索排序能力，让 AI 和用户检索文档时更容易命中相关内容。
- 新增移动到文件夹相关 UI 能力，整理文档更方便。
- 补充云盘文件夹同步方案设计，为后续基于外部文件夹的同步能力打基础。

## 修复内容

- 修复 Agent 主循环没有真正使用任务执行器的问题。
- 修复工具失败后任务仍继续运行、进度状态不准确的问题。
- 修复停止 Agent 后任务进度仍显示执行中的问题。
- 修复创建文档提示词过度追问的问题。
- 修复多轮创建文档补充信息时，原始需求和中间偏好丢失的问题。
- 修复 AI 输出、工具结果和完成判断之间状态不同步的问题。
