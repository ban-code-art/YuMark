## ❌ 高优先级

### F-1 系统返回键/手势不触发保存（编辑器）

- [x] 已修复（2026-06-12）

**位置**：`EditorScreen.kt`

**问题**：此前修复的保存竞态只覆盖了顶栏返回箭头（`saveAndWait` 后导航）。系统返回手势/返回键走默认导航，不经过任何保存逻辑，ViewModel 直接销毁——自动保存间隔内（默认 30 秒）的修改**静默丢失**。另外顶栏箭头快速双击会触发两次 `navigateUp` 连退两页。

**修复方案**：统一出口——`BackHandler` 与顶栏箭头共用同一个带防抖标记的 `saveAndExit` 逻辑。

## ⚠️ 中优先级

### F-2 "打开抽屉自动重扫工作区"未实现（工作区）

- [x] 已修复（2026-06-12）

**位置**：`FileListScreen.kt`

**问题**：设计文档承诺"打开左侧抽屉时自动 rescan + 手动刷新按钮"，实际只有手动刷新。外部新增的文件必须手点刷新才出现。

**修复方案**：`LaunchedEffect(drawerState.isOpen)`：抽屉打开且工作区存在时调 `rescanWorkspace()`。

### F-3 工作区模式下主界面/FAB/搜索语义混乱（工作区 + 搜索）

- [x] 已修复（2026-06-12）

**位置**：`FileListScreen.kt`

**问题**：打开外部文件夹后，侧栏是工作区文件树，但主界面仍是内部文档库列表（常为空）、FAB 新建的是内部文档、搜索搜的也是内部库——与用户"我在浏览这个文件夹"的心智模型冲突。

**修复方案**（短期）：工作区模式下主界面改为提示卡片（显示文件夹名 + "从侧栏打开文档" + 打开侧栏按钮），隐藏 FAB 与搜索/排序入口。长期可让主列表与搜索切换到工作区文档（另立需求）。

### F-4 工具栏行级语法插入位置不对（工具栏）

- [x] 已修复（2026-06-12）

**位置**：`EditorScreen.kt`（`MarkdownToolbar` 的 `onInsertSyntax` 回调）

**问题**：标题 `# `、列表 `- `/`1. `、引用 `> ` 是行级语法，当前插在光标处——光标在行中间时产生 `文字# ` 这类坏 Markdown。

**修复方案**：行级语法插入到当前行行首（`lastIndexOf('\n', pos-1) + 1`），包裹型语法维持光标处插入。

### F-5 HTML 导出文件名未消毒（导出）

- [x] 已修复（2026-06-12）

**位置**：`core/export/HtmlExporter.kt`（`export` 直接用 `document.name` 拼文件名）

**问题**：`sanitizeFileName` 只用在 Markdown 导出路径；HTML 路径未消毒。目前创建文档时有 `FileNameValidator` 把关所以实际风险低，但防线不一致——绕过创建校验的名字（如导入的外部文件名）会在导出时出问题。

**修复方案**：`FileNameValidator` 增加 `sanitize(name)`，`HtmlExporter` 使用之（后续 `ExportDocumentUseCase` 的私有实现也可收敛过来）。

### F-6 设置页存在"假设置"（设置）

- [x] 已修复（2026-06-12）

**位置**：`SettingsScreen.kt`、`EditorScreen.kt`、`EditorViewModel.kt`

**问题**：
- "字体大小"滑块只存值：编辑区用固定的 `bodyLarge`，预览 WebView 也不注入字号——调了没有任何效果
- "自动压缩图片"行没有开关控件，纯展示（图片功能本身休眠）

**修复方案**：字体大小真正生效——编辑区 `textStyle` 用 `settings.fontSize.sp`（行高 1.6 倍同步），预览注入 `body{font-size}`；"自动压缩图片"行先移除（待图片功能接线时一并恢复）。
