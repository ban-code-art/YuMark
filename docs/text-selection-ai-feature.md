# 文本选择 AI/Agent 快捷功能设计方案

## 功能概述
在编辑器中选中文本后，可以直接在当前界面询问 AI 或使用 Agent，无需跳转到 AI 功能界面。

## 交互流程

### 1. 选中文本
```
用户选中文本：
"Kotlin 是什么？"
    ↓
显示悬浮菜单：
[💬 询问 AI]  [🤖 Agent 处理]
```

### 2. 点击按钮
```
点击"询问 AI" → 弹出输入框 → 输入问题 → 显示回复（只读）
点击"Agent 处理" → 弹出输入框 → 输入需求 → Agent 修改文本
```

### 3. AI 回复界面
```
┌─────────────────────────────┐
│ 💬 询问 AI                  │
├─────────────────────────────┤
│ 关于：                       │
│ "Kotlin 是什么？"           │
│                             │
│ 你的问题：                   │
│ ┌─────────────────────────┐ │
│ │ 请解释这段内容           │ │
│ └─────────────────────────┘ │

```

## UI 组件设计

### 1. 文本选择菜单
- 悬浮在选中文本上方
- 两个按钮：询问 AI、Agent 处理
- 自动定位，不遮挡文本

### 2. 对话框样式
- ModalBottomSheet 形式
- 固定在底部，高度 60% 屏幕
- 支持滑动关闭

### 3. 实时反馈
- AI 回复：流式输出，Markdown 渲染
- Agent：显示工作动画，完成后显示建议

## 技术实现

### 数据流
```
EditorScreen
    ↓ (用户选中文本)
    selectedText: String
    ↓ (点击按钮)
    showAiQuickDialog = true
    ↓ (调用 AI/Agent)
    AiQuickDialog → ViewModel → UseCase → AI API
    ↓ (显示结果)
    Result (AI: 只读显示，Agent: 可应用)
```

### 组件结构
```
EditorScreen.kt
├─ TextEditor (编辑器)
├─ SelectionMenu (悬浮菜单)
└─ AiQuickDialog (对话框)
    ├─ AiQueryMode (询问 AI)
    └─ AgentMode (Agent 处理)
```

## 优势

### 用户体验
- ✅ 无需跳转，在当前界面完成
- ✅ 保持编辑上下文
- ✅ 快速完成简单任务

### 功能区别
| 功能 | 完整 AI 界面 | 快捷功能 |
|------|-------------|---------|
| 对话历史 | ✅ | ❌ |
| 多轮对话 | ✅ | ❌ |
| 文档关联 | ✅ | ✅ |
| Agent 操作 | ✅ | ✅ (仅选中文本) |
| 使用场景 | 深度对话 | 快速任务 |

## API 设计

### AiQuickDialog
```kotlin
@Composable
fun AiQuickDialog(
    selectedText: String,
    mode: QuickMode,  // AI_QUERY / AGENT_EDIT
    onDismiss: () -> Unit,
    onApplyEdit: (String) -> Unit  // Agent 模式下应用修改
)
```

### QuickMode
```kotlin
enum class QuickMode {
    AI_QUERY,    // 询问 AI（只读）
    AGENT_EDIT   // Agent 处理（可应用）
}
```
