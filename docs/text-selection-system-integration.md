# 文本选择 AI 功能 - 系统集成方案

## 当前问题
1. 按钮在应用内部显示（底部栏）
2. 没有选中文本时仍然显示
3. 不够原生，不融入系统菜单

## 目标效果
将 AI 选项添加到 Android 系统文本选择菜单中：

```
┌──────────────────────────────────────┐
│ 🌐 Translate  📋 Copy  🔗 Share  ⋮  │
│ 💬 询问 AI   🤖 Agent              │  ← 集成到系统菜单
└──────────────────────────────────────┘
```

## 技术方案

### 方案 A: ActionMode (推荐)
- 使用 `android.view.ActionMode`
- 添加自定义菜单项
- 完全原生外观

### 方案 B: 悬浮菜单
- 使用 `PopupWindow`
- 定位在选中文本上方
- 模拟系统样式

### 方案 C: 简化方案
- 保持底部按钮
- 但只在**真正有选中文本**时显示
- 优化按钮样式

## 建议
先实现**方案 C**（简化），因为：
1. 方案 A 需要修改 WebView 的 ActionMode，很复杂
2. 方案 B 需要精确计算位置
3. 方案 C 最快实现，体验也不错

## 下一步
1. 修复"没有选中文本时仍显示按钮"的问题
2. 优化按钮样式，更接近系统原生
