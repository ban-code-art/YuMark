# 编辑器返回键优化 - 一键退出

**问题**: 在预览模式下按返回键，先切换到编辑模式，再按一次才能退出

**日期**: 2026-06-15

---

## 🎯 问题描述

### 旧行为

用户在预览模式下的退出流程：

```
预览模式 → 按返回键 → 切换到编辑模式 → 按返回键 → 退出
  ↑                       ↓
  |                   还需要再按一次
  └───────────────────────┘
```

**体验问题**:
- ❌ 需要按两次返回键才能退出
- ❌ 中间会闪过编辑界面
- ❌ 不符合用户预期

---

## ✅ 优化方案

### 新行为

无论编辑模式还是预览模式，一次返回键直接退出：

```
预览模式 → 按返回键 → 自动保存 → 退出
编辑模式 → 按返回键 → 自动保存 → 退出
  ↑                                  ↓
  └──────────────一次退出─────────────┘
```

**优势**:
- ✅ 一次返回键直接退出
- ✅ 自动保存文档，不丢失内容
- ✅ 符合用户直觉

---

## 🔧 技术实现

### EditorScreen.kt - BackHandler 优化

**旧代码**:
```kotlin
// 返回键处理优先级（从高到低）：
// 1. 抽屉打开时先收抽屉
BackHandler(enabled = fileDrawerState.isOpen) {
    scope.launch { fileDrawerState.close() }
}

// 2. 预览模式下返回到编辑模式
BackHandler(enabled = isPreviewMode) {
    viewModel.togglePreviewMode()  // ← 先切换到编辑模式
}

// 3. 编辑模式下返回键退出并保存
var isExiting by remember { mutableStateOf(false) }
BackHandler(enabled = !isPreviewMode && !fileDrawerState.isOpen) {
    if (!isExiting) {
        isExiting = true
        scope.launch {
            viewModel.saveAndWait()
            navController.navigateUp()
        }
    }
}
```

**问题分析**:
- `BackHandler` 的 `enabled` 条件导致优先级冲突
- 预览模式下，第一个 `BackHandler(enabled = isPreviewMode)` 被触发
- 只是切换模式，没有退出

**新代码**:
```kotlin
// 返回键处理优先级（从高到低）：
// 1. 抽屉打开时先收抽屉
BackHandler(enabled = fileDrawerState.isOpen) {
    scope.launch { fileDrawerState.close() }
}

// 2. 预览模式或编辑模式下返回键直接退出并保存
var isExiting by remember { mutableStateOf(false) }
BackHandler(enabled = !fileDrawerState.isOpen) {  // ← 简化条件
    if (!isExiting) {
        isExiting = true
        scope.launch {
            viewModel.saveAndWait()  // 自动保存
            navController.navigateUp()  // 退出
        }
    }
}
```

**优化要点**:
- ✅ 移除预览模式的单独处理
- ✅ 统一为一个 `BackHandler`
- ✅ 只要抽屉关闭，返回键就直接退出
- ✅ 退出前自动保存，防止数据丢失

## 🔒 安全机制

### 防抖处理

```kotlin
var isExiting by remember { mutableStateOf(false) }
BackHandler(enabled = !fileDrawerState.isOpen) {
    if (!isExiting) {  // ← 防止重复触发
        isExiting = true
        scope.launch {
            viewModel.saveAndWait()  // 异步保存

```

**作用**:
- 防止用户快速连按返回键
- 避免重复保存和导航
- 确保退出流程完整执行

### 自动保存

```kotlin
viewModel.saveAndWait()  // 等待保存完成
navController.navigateUp()  // 再退出
```

**保证**:
- ✅ 所有修改都会保存
- ✅ 不会丢失用户数据
- ✅ 即使用户快速退出也安全

## 💡 设计思路

### 简化状态机

**旧逻辑**:
```
状态机：编辑模式 ⇄ 预览模式 → 退出
         ↑_____返回键_____↑   ↑
                              |
                          返回键（需要两次）
```

**新逻辑**:
```
状态机：编辑模式 ⇄ 预览模式
         ↓           ↓
         └─ 返回键 ──┘
               ↓
            退出（一次）
```

### 原则

1. **用户意图优先**: 返回键表达的是"我想离开"，而不是"我想切换模式"
2. **最小步骤**: 减少不必要的中间状态
3. **安全第一**: 退出前自动保存，防止数据丢失
