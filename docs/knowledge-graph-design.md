## 1. 功能概述

### 1.1 功能描述

知识图谱是一个可视化的文档关系网络，以**图**的形式展示所有文档之间的链接关系：
- **节点（Node）**：每个文档是一个节点
- **边（Edge）**：文档之间的链接关系
- **交互**：点击节点打开文档，拖拽调整位置，缩放查看细节

### 1.2 核心价值

1. **全局视角** - 一眼看清所有文档的组织结构
2. **发现联系** - 直观展示文档间的引用关系
3. **知识探索** - 从一个主题跳转到相关主题
4. **思维导图** - 可视化个人知识体系

### 1.3 使用场景

- 📚 构建个人知识库，查看主题关联
- 🔍 发现孤立文档（没有任何链接的文档）
- 🌐 探索文档网络，跟随链接深入研究
- 📊 统计分析：哪些文档是中心节点（被引用最多）

## 3. 交互设计

### 3.1 基础交互

| 操作 | 效果 |
|------|------|
| **单击节点** | 高亮该节点及其直接连接 |
| **双击节点** | 打开该文档编辑器 |
| **长按节点** | 固定/取消固定节点位置 |
| **拖拽节点** | 调整节点位置 |
| **拖拽画布** | 平移视图 |
| **双指缩放** | 放大/缩小整个图谱 |
| **悬停节点** | 显示文档信息：标题、字数、最后修改时间 |

### 3.2 搜索与过滤

**顶部搜索框**：
- 输入文档名，高亮匹配节点
- 自动聚焦到搜索结果

**过滤选项**（设置菜单）：
- ☑️ 显示孤立文档
- ☑️ 仅显示标签为 #重要 的文档
- ☑️ 隐藏小于 100 字的文档
- ☑️ 仅显示最近 7 天修改的文档

### 3.3 布局算法

**力导向布局（Force-Directed Layout）**：
- 节点之间相互排斥（避免重叠）
- 有连线的节点相互吸引（聚集相关内容）
- 自动达到视觉平衡

**可选布局**：
- 🌐 力导向（默认）
- 🌳 树形布局
- ⚪ 环形布局
- 📐 层次布局

## 4. 技术实现

### 4.1 技术选型

**Android Canvas 自定义绘制 vs Web 图表库**

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Canvas 自绘** | 性能好，完全控制 | 需要手写布局算法 |
| **WebView + D3.js** | 成熟库，开箱即用 | 性能稍低，文件体积大 |
| **Jetpack Compose Canvas** | 现代化，易维护 | 需要手写图算法 |

**推荐方案**：**WebView + D3.js**
- 成熟的力导向布局算法
- 丰富的交互能力
- 快速迭代开发

**备选方案**：**Compose Canvas + 自研算法**
- 性能更优，适合大规模图谱（1000+ 节点）
- 完全原生体验

### 4.2 数据模型

#### 图谱数据结构
```kotlin
/**
 * 图谱节点
 */
data class GraphNode(
    val id: String,              // 文档 ID
    val label: String,           // 显示名称
    val size: Int = 1,           // 节点大小（基于引用次数）
    val color: String? = null,   // 节点颜色（基于文件夹/标签）
    val group: String? = null,   // 分组标识
    val x: Float? = null,        // 位置 X（固定位置时使用）
    val y: Float? = null         // 位置 Y
)

/**
 * 图谱连线
 */
data class GraphEdge(
    val source: String,          // 源文档 ID
    val target: String,          // 目标文档 ID
    val type: LinkType = LinkType.UNIDIRECTIONAL,  // 链接类型
    val weight: Int = 1          // 权重（引用次数）
)

enum class LinkType {
    UNIDIRECTIONAL,  // 单向链接 A → B
    BIDIRECTIONAL    // 双向链接 A ↔ B
}

/**
 * 完整图谱
 */
data class KnowledgeGraph(
    val nodes: List<GraphNode>,
    val edges: List<GraphEdge>,
    val isolated: List<GraphNode>  // 孤立节点（无任何连接）
)
```

### 4.3 链接解析

**Markdown 链接语法支持**：

1. **Wiki 链接**：`[[文档标题]]`
2. **Markdown 链接**：`[显示文本](文档标题.md)`
3. **相对路径**：`[文本](../folder/doc.md)`

**解析流程**：
```
遍历所有文档
  ↓
正则匹配链接语法
  ↓
提取目标文档名/路径
  ↓
查找对应文档 ID
  ↓
构建 GraphEdge
```

**正则表达式**：
```kotlin
// Wiki 链接
val wikiLinkRegex = """\[\[([^\]]+)\]\]""".toRegex()

// Markdown 链接（文档内部链接）
val mdLinkRegex = """\[([^\]]+)\]\(([^)]+\.md)\)""".toRegex()
```
