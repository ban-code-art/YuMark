# 创建 GitHub Release v0.6.1 指南

## 方法 1：使用 GitHub 网页界面

1. 访问：https://github.com/ban-code-art/YuMark/releases/new

2. 填写以下信息：

**Tag version**: `v0.6.1`

**Release title**: `v0.6.1 - AI 渲染、热更新、自动推送更新`

**描述**：
```markdown
## 🎉 新功能

### 1. AI 消息 Markdown 渲染
- AI 助手输出的内容现在会自动渲染为 Markdown 格式
- 支持标题、列表、代码块、表格、引用等语法
- 自动适配深色/浅色主题

### 2. 文档热更新机制
- AI 编辑文档后，编辑器会自动刷新显示最新内容
- 无需手动切换文档即可看到更改
- 提升 AI 辅助编辑的流畅性

### 3. 优化编辑器顶部工具栏
- 简化工具栏布局，保留核心功能
- 将次要功能（保存、导出、AI、设置）收纳到"更多"菜单
- 界面更简洁，减少视觉压力

### 4. 启动时自动检查更新 ⭐
- 应用启动时自动在后台检查新版本
- 发现更新时智能弹窗提示
- 支持"稍后提醒"和"立即更新"选项
- 无更新或检查失败时静默处理，不打扰用户

## 🐛 Bug 修复

- 修复 AI 输出内容显示为纯文本的问题
- 修复 AI 编辑后需要手动刷新才能看到更改的问题

## 📥 下载

请下载下方的 **YuMark-v0.6.1.apk** 文件进行安装。

**安装要求**：
- Android 8.0 (API 26) 及以上
- 约 4 MB 存储空间

**注意**：如果已安装旧版本，可以直接覆盖安装。

---

**完整更新日志**: [CHANGELOG_v0.6.1.md](../CHANGELOG_v0.6.1.md)
```

3. **上传 APK 文件**：
   - 拖拽 `YuMark-v0.6.1.apk` 到附件区域
   - 等待上传完成

4. **设置为最新版本**：
   - ✅ 勾选 "Set as the latest release"

5. 点击 **Publish release**

## 更新检查 API 说明

应用会从以下 URL 获取最新版本信息：

```
https://api.github.com/repos/ban-code-art/YuMark/releases/latest
```

返回的 JSON 包含：
- `tag_name`: 版本号（如 "v0.6.1"）
- `body`: 更新日志
- `published_at`: 发布时间
- `assets[0].browser_download_url`: APK 下载链接
- `assets[0].size`: 文件大小

应用会自动解析并与当前版本（BuildConfig.VERSION_NAME）比较。

## 故障排除

### 问题 1：启动时没有弹出更新提示
**可能原因**：
- GitHub Release 未创建或未设为最新
- 网络连接问题
- API 请求失败

**解决方法**：
1. 检查 Logcat 日志：`adb logcat | grep "FileListViewModel"`
2. 查看是否有 "启动时检查更新失败" 的日志
3. 手动访问 API 确认 Release 存在：
   ```bash
   curl https://api.github.com/repos/ban-code-art/YuMark/releases/latest
   ```

### 问题 2：下载失败
**可能原因**：
- GitHub 服务器连接问题
- 文件过大，下载超时

**解决方法**：
- 重试下载
- 检查网络连接

### 问题 3：安装失败
**可能原因**：
- APK 签名不一致
- 存储空间不足

**解决方法**：
- 确保使用相同的签名密钥构建两个版本
- 检查手机存储空间
